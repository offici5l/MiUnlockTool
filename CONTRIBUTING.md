# Contributing

All pull requests are welcome, no matter how small — code, documentation, or anything else.

## Documentation

- ​Write clearly for everyone whether they are technical or not
- ​Organize the documentation to avoid distraction

## Code

- Make sure your changes don't negatively affect other operating systems
- Keep the code clean and concise where possible
- For large changes, split into separate files to keep things maintainable

## Discussing Ideas

If you have an idea but are hesitant to work on it, feel free to open an issue or [reach out](mailto:offici5lcontact@proton.me) to discuss it first

> [!NOTE]
> If your PR isn't merged, it's not a reflection of your work.
> It simply means it doesn't align with the vision for this project.
